Natours – Adventure Tour Booking App

A full-stack web application for booking and managing adventure tours.

Natours is a complete web application built as part of Jonas Schmedtmann’s Node.js course, showcasing modern full-stack development concepts. The platform enables users to explore available adventure tours, view details, sign up or log in, and securely book tours with integrated online payments.

The app includes user authentication and authorization using JWT, an admin dashboard for managing tours and users, and a secure booking system through the Stripe API. Each tour features detailed information such as duration, price, and difficulty, presented in a visually appealing, responsive design.

This project demonstrates practical experience with API design, data modeling, and server-side rendering using Pug. It also incorporates important backend concepts like security (Helmet, data sanitization, rate limiting) and deployment through Heroku and MongoDB Atlas.

  • Technologies: Node.js, Express, MongoDB, Pug, Stripe API
  • Category: Web Application
  • Github Link: https://github.com/TsinadzeNikoloz/Natours

Mziuri Reactional API

A modular and scalable RESTful backend built with Node.js, Express, and MongoDB.

Mziuri Reactional API is a backend-focused project developed to provide a clean, modular, and extensible RESTful architecture for a multi-functional web platform. The API supports multiple domains — from authentication and content management to e-commerce-like operations — all built with scalability and structure in mind.

It features user authentication and role-based authorization using JSON Web Tokens (JWT), allowing both user and admin access levels. The system includes routes for managing products, orders, menus, and categories, along with blogging functionality through posts and comments. It also supports an album and photo upload module for organizing media.

Built following MVC principles, the project uses validation middleware, centralized error handling, and a well-structured folder system (controllers, routes, models, and validators). All routes are protected with authentication and role-based guards, ensuring data integrity and secure access to resources.

  • Technologies: Node.js, Express, MongoDB, JWT, Mongoose
  • Category: RESTful API / Backend Development
  • Github Link: https://github.com/TsinadzeNikoloz/MembersAPI

Borderland Survival Game – Java Simulation

A Java-based survival game simulation demonstrating advanced OOP and design patterns.

A simulation of a strategic survival game where characters with unique stats and personalities face mental and physical challenges to win. Built with the Strategy pattern for dynamic behavior changes and a state machine for player decisions, it models trust, betrayal, and cooperation among players.

The system includes clear modular components (GameMaster, Arena, BorderlandManager), file-driven initialization, and a well-documented design supported by UML class, sequence, and state diagrams. Comprehensive JUnit tests ensure consistent gameplay and logic validation.

  • Technologies: Java, OOP, Strategy Pattern, JUnit 4
  • Category: Simulation / Object-Oriented Design
  • Github Link: https://github.com/TsinadzeNikoloz/BorderlandGame

Desert Rails – Logic Puzzle Game

A browser-based logic game where players build rail networks to escape the desert.

Desert Rails is an interactive puzzle game where players place rail pieces on randomly generated maps featuring bridges, mountains, and oases. The goal is to create a valid, connected rail network that allows the train to escape the desert before time runs out.

The game includes dynamic map generation with easy and hard modes, click-based tile placement with real-time rendering, automatic solution validation, and a difficulty-based leaderboard stored in Local Storage so results persist between sessions.

  • Technologies: HTML, CSS, JavaScript
  • Category: Browser Game / Logic Puzzle
